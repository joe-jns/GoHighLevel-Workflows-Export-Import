## What this is

A Chrome MV3 extension that adds **Export / Import** capabilities to the
GoHighLevel **Workflows** page. It works on any GHL tenant, including
white-labeled ones, because it targets the underlying iframe SPA origin
(`client-app-automation-workflows.leadconnectorhq.com`) rather than the
wrapper domain.

User-facing docs: `README.md`.
API recon + schemas: `docs/ghl-recon.md`.
Design spec: `docs/superpowers/specs/2026-05-19-ghl-workflow-exporter-design.md`.

## File map

| File | Runs in | Role |
|---|---|---|
| `manifest.json` | — | MV3 declaration. `host_permissions` covers the iframe SPA origin + `backend.leadconnectorhq.com` for active fallback fetches. |
| `page-hook.js` | **MAIN world** (page JS context) | Monkeypatches `window.fetch` and `XMLHttpRequest` to clone workflow API responses and dispatch them as `CustomEvent('ghl-export:capture')`. Also forwards the `token-id` header it sees. |
| `content.js` | **Isolated content-script world** | The orchestrator. Listens for capture events, maintains the in-memory cache, reads/refreshes the auth token, injects the toolbar button, renders the dropdown menu and import-selection modal, runs the read/write API calls, and hands payloads to the service worker. |
| `background.js` | **Service worker** | Builds ZIPs (vendored JSZip), serializes JSON, performs `chrome.downloads.download` via data: URLs, and parses uploaded ZIPs back into individual JSON envelopes. |
| `styles.css` | content world | Button, dropdown, toast, and the import-selection modal. |
| `lib/jszip.min.js` | service worker | Vendored ~100 KB. `importScripts(...)` in `background.js`. |
| `icons/{16,48,128}.png` | — | Generated by `scripts/make-icons.js` (Node, no deps). Indigo solid placeholders. |
| `scripts/make-icons.js` | Node | Hand-rolls minimal PNGs with crc32. Run only when refreshing icons. |
| `docs/recon-samples/` | — | **gitignored.** Live API payloads captured during recon. Personally identifiable. |

## Architecture cheat-sheet

```
white-label wrapper page (any domain)
└── iframe[name=workflow-builder]   ← extension content_script matches HERE
    SPA origin: client-app-automation-workflows.leadconnectorhq.com
    ├── page-hook.js  ── intercepts fetch/XHR ──┐
    │                                           │ CustomEvent('ghl-export:capture')
    └── content.js  <───────────────────────────┘
        ├── in-memory cache  Map<locId, {folders, workflows, latestToken}>
        ├── injects "Export / Import ▾" button next to "Create Workflow"
        ├── direct fetches to backend.leadconnectorhq.com (token-id header)
        └── chrome.runtime.sendMessage → service worker
                                          ↓
                                       background.js
                                       ZIP build / file download
```

## Auth model

- The GHL SPA uses a Firebase JWT stored as `localStorage.refreshedToken`
  (JSON-stringified). It rotates every ~hour; the SPA refreshes it
  automatically on page load.
- `page-hook.js` also captures the `token-id` from any outgoing request
  header, so we always have a fresh value even mid-session.
- Active-fallback fetches re-attach `token-id` + `channel: APP` headers.
- **The extension hardcodes ZERO user-specific IDs.** Everything is derived
  at runtime from the URL (`/location/{locId}/...`) or the JWT payload
  (decoded for `userId` and `companyId` in `decodeJwtUserInfo()`).

## API endpoints used

Reads (passive + active):
- `GET /workflow/{locId}/list?parentId=…|type=workflow` — folders + workflows
- `GET /workflow/{locId}/{workflowId}?…` — workflow detail (actions in `workflowData.templates[]`)
- `GET /workflow/{locId}/trigger?workflowId=…` — triggers

Writes (import):
- `POST /workflow/{locId}` — create empty workflow → `{id}`
- `POST /workflow/{locId}/directory` — create folder → `{id}`
- `POST /workflow/{locId}/trigger` — create trigger → `{id}`
- `PUT  /workflow/{locId}/{wfId}/auto-save` — push `workflowData.templates[]` + `createdSteps[]` + `newTriggers[]`

Full schemas live in `docs/ghl-recon.md`.

## Non-obvious things you'll trip on

1. **GHL list checkboxes are NaiveUI custom components**, not native
   `<input type="checkbox">`. Selector: `.n-checkbox--checked` or
   `[role="checkbox"][aria-checked="true"]`. Header master checkbox is
   excluded by scoping to `td.n-data-table-td--selection`.
2. **Workflow/folder UUIDs live in `td[data-col-key="name"] a[id]`** — not
   in an href. Same DOM shape for workflows and folders; differentiate
   by looking up the UUID in our cache (`L.workflows` vs `L.folders`).
3. **Folder selection auto-expands.** `resolveSelectedToWorkflowIds()`
   walks folders recursively via `GET /list?parentId=…` and returns a flat
   list of workflow IDs.
4. **Service workers can't use `blob:` URLs for `chrome.downloads`.**
   `background.js` encodes the blob as a `data:` URL via `blobToDataUrl`.
5. **Triggers are saved separately from steps**, in this order during import:
   create workflow shell → POST each trigger → PUT auto-save with steps and
   `newTriggers[]` referencing the freshly-assigned trigger IDs.
6. **Cross-tenant ID drift is real.** Imported workflows keep their source
   step IDs and any referenced `pipelineId`/`calendarId`/`customFieldId`.
   Those won't exist in the target tenant — the workflow saves but those
   steps need manual fixing. Documented in README's caveats.
7. **No build step.** Three content-script files are loaded directly by
   the manifest (`content_scripts.js: ["content.js"]`); the page hook is
   shipped via `web_accessible_resources` and injected at runtime by
   `content.js`. JSZip is loaded into the service worker via
   `importScripts('lib/jszip.min.js')`.

## How to develop

1. Edit a file.
2. Go to `chrome://extensions`, hit the reload icon on the extension card.
3. Reload any open GHL workflow page so the new content script loads.
4. Check DevTools:
   - Page console (inside iframe) shows content script + page hook logs.
   - `chrome://extensions` → "service worker" link opens the BG worker console.

`node --check content.js background.js page-hook.js scripts/make-icons.js`
catches syntax errors without running the extension.

## How to add a new export field

If you need to ship a new field per workflow in the JSON envelope:

1. Capture it (passively or actively) — extend `handleCapture()` in `content.js`
   or add an active-fetch helper alongside `fetchWorkflowDetail()`.
2. Add it to `envelopeSingle()` and `envelopeCombined()`.
3. Update `docs/ghl-recon.md` "Export schema" section.
4. Bump the schema version if it's a breaking change (`$schema: ghl-workflow-export/v2`).

## How to add a new write endpoint

1. Capture it in Playwright (MCP `browser_network_request` with `request-body`).
2. Document the payload shape in `docs/ghl-recon.md` "Write endpoints".
3. Add a new function next to `createEmptyWorkflow`/`createTrigger`/`createFolder`
   in `content.js`, using `authedFetch(url, 0, {method, headers, body})`.
4. Wire it into `importWorkflowEnvelope()`.

## Recon workflow (when something breaks or GHL changes the API)

This project was built recon-first: the extension only knows endpoints that
were observed firing in real Playwright sessions. To re-do recon:

1. Launch Playwright MCP, navigate to the workflows page.
2. Perform the action you care about (create, save, copy, etc.).
3. `browser_network_requests` with a filter on `workflow|trigger|directory`.
4. `browser_network_request` with `part: "request-body"` and `part: "response-body"` on the relevant index.
5. Save the captured JSON to `docs/recon-samples/` (gitignored — they contain
   PII and Firebase signed URLs).
6. Update `docs/ghl-recon.md` with a sanitized schema.

## Things deliberately not in scope (v1)

- Canvas/layout positions (the Firebase `fileUrl` snapshot is ignored)
- Sticky notes (`/workflows/sticky-notes-all`)
- Per-step enrollment stats
- Cross-tenant ID remapping (pipeline/calendar/customField IDs)
- Firefox/Safari (MV3 Chrome/Edge only)
- Nested folder support beyond what `ensureFolders()` already does
  (topo-sort works, but the export side currently doesn't reliably walk
  ancestor chains — see TODO if/when nested folders matter)

## Quick sanity check

If a user reports "Export selected does nothing":
- Confirm GHL hasn't changed `td[data-col-key="name"] a[id]` (most likely culprit).
- Inspect a row in DevTools; if the structure changed, update `readSelectedIds()`.

If "No auth token captured":
- The user hasn't loaded a workflow in this session yet, so
  `localStorage.refreshedToken` is set but `page-hook.js` hasn't seen a
  request with the live header. `bootstrapToken()` already covers this;
  if still failing, the localStorage key name probably changed.
