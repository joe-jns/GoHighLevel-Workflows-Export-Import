# GHL workflow primitives catalog

Generated by `scripts/build-catalog.js` from a live capture of the GHL
`/workflows-marketplace/location/{locId}/assets` endpoint.

## Files

| File | Purpose |
|---|---|
| `ghl-workflow-catalog.json` | **Master reference.** Single JSON containing all actions and triggers, indexed by their `key` (the workflow `type` value). Hand this to Claude when asking it to generate workflows. |
| `ghl-workflow-catalog.md` | Human-readable index, grouped by app, one line per primitive. Browse with your eyes. |
| `actions.json` | Subset with only actions, same shape. |
| `triggers.json` | Subset with only triggers, same shape. |

## Schema (per action)

```json
{
  "key": "add_contact_tag",
  "name": "Add Contact Tag",
  "app": "contact",
  "description": "...",
  "icon": "fa-tag",
  "executionType": "INTERNAL",
  "inputs": [
    { "field": "tags", "label": "Tags", "type": "tag_input", "required": true,
      "options": [ ... ], "default": ..., "mappedTo": "TAGS",
      "dynamicFrom": { "service": "...", "route": "..." } }
  ]
}
```

## Schema (per trigger)

```json
{
  "key": "contact_created",
  "name": "Contact Created",
  "app": "contact",
  "description": "...",
  "type": "INTERNAL",
  "filters": [ { "field": "...", "label": "...", "type": "...", "required": ... } ],
  "contextVariables": ["CONTACT"]
}
```

## Rebuild

```bash
node scripts/build-catalog.js
```

## Sources

Entries come from two places, marked on each entry via `source`:

- `marketplace-assets`: returned by
  `GET /workflows-marketplace/location/{locId}/assets`. Richest input schemas
  (full `inputs[]` with field types, options, validations). Covers most
  third-party integrations (Linear, Asana, etc.) and many native actions.
- `spa-bundle`: extracted from the workflows SPA bundle
  (`assets/index-DrASS8UJ.js`) which holds the truly-internal builder
  primitives (If/Else, Wait, Drip, Webhook, Custom Code, etc.) and core
  triggers (opportunity, shopify, courses, IVR). For these, `inputs[]` is
  empty because each type uses a bespoke Vue config component. To learn
  their `attributes` shape, see the `example` field on each entry (populated
  from captured workflow samples) or export real workflows that use them.

The merge happens in `scripts/merge-bundle-into-catalog.js`. To refresh:

```bash
node scripts/download-bundle.js          # refresh bundle-cache/
node scripts/build-catalog.js            # base catalog from assets capture
node scripts/merge-bundle-into-catalog.js  # add bundle primitives
```

The `download-bundle.js` script reads its asset list inline — you may need
to update the list of chunks to match what the current SPA loads (open
DevTools Network panel, filter for `client-app-automation-workflows`).

## Re-capture from a fresh GHL session

Re-capture the source data with the browser console script at
`scripts/dump-primitives-catalog.js`, then put the contents of the
`assets` field into `docs/recon-samples/cat-assets.json` and rerun.
